Project Showcase

1. Laser communication device

Led team of three students that built a laser transceiver capable of transmission rates of up to 0.7kB/s, close to the theoretically determined limit. The project led to an invitation for presentation at IEMCON 2016.

The device was built using an inexpensive laser emitter and a photodiode (combined price <1$) and an Arduino. Initially, we built a prototype and tested the idea for feasibility which was successful. Following that, we built the actual devices and then wrote our own communication protocol. The device is able to automatically detect start-message and end-message signatures due to a predetermined baud rate agreed upon by both the sending and receiving device. It also includes robust error detection, such as the detection of interrupted messages and improper message encoding.

As a user interface, we developed a Meteor app which serves as the chat server and provides a GUI (seen in image above). The chat was reliable over distances of up to 10m. Entire paragraphs could be sent in less than the blink of an eye. The source code is available on github at https://github.com/sbambey/DialUp. The final report is available here.

2. Antenna tracker for Unmanned Aircraft Systems

As part of the Unmanned Aircraft Systems (UAS) Engineering Design Team at UBC, I developed the electronics and software and assisted with the mechanical build for an antenna tracker. The tracker software interfaces with the drone using a protocol called Mavlink and receives live telemetry. It then calculates a yaw and a pitch angle for the antenna and drives two stepper motors in the appropriate directions. The motors were initially controlled by two inexpensive stepper motor drivers that gave us some trouble. The team then received a high-end stepper motor controller which I integrated into the tracker. Feedback on the actual position of the antenna is provided by an Inertial Measurement Unit (IMU) mounted to the antenna plate. Working with the IMU has been a challenge, as drift in the output persists to be an issue. In the future, I am planning to replace the IMU with a mechanical feedback that does not have the drift issues.

3. flightcrew.io

flightcrew.io is a job board written in Ruby on Rails, that automatically matches pilots with jobs appropriate for their experience level. The functionality includes a detailed sign up system which allows users to enter their qualifications such as flight hours, licenses, ratings and more. A comprehensive matching algorithmn then matches pilots with jobs they are eligible for. Furthermore, there is an extensive back-end for adding jobs and finding updates to job postings on external sites. The source code is available on github at https://github.com/sbambey/flightcrew and a live demo can be seen at https://safe-forest-2459.herokuapp.com/ (may take a few seconds to load).

4. manifest.space

manifest.space is a log of some of SpaceX's launches. It is a passion project for me and I occassionally find time to update it. The project is a Ruby on Rails app and is a great way for me to fiddle with different libraries and tools. In the early phases, I collaborated with some other people over the internet and as a result, I developed a basic revision tracking system available through the admin panel and integrated a markdown editor. The source code is available on github at https://github.com/sbambey/manifest and a live demo can be seen at http://www.manifest.space (may take a few seconds to load).
